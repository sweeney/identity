1. Executive Summary

We identified 13 findings across the codebase and live service. The most critical is a wildcard CORS misconfiguration that allows any website to make authenticated API requests and read responses. Combined with the absence of rate limiting and a timing side-channel, an attacker can enumerate users, brute-force passwords, and exfiltrate data cross-origin.

The codebase demonstrates strong security fundamentals overall — parameterized SQL, hashed tokens, PKCE enforcement, token family rotation — making these findings stand out as fixable gaps rather than systemic issues.

2. Vulnerability Findings

HIGH Wildcard CORS Origin Reflection CONFIRMED LIVE
cmd/server/main.go:337-345

The CORS middleware reflects any Origin header value back in Access-Control-Allow-Origin, effectively disabling the Same-Origin Policy for all API and OAuth token endpoints.

Vulnerable Code

// CORS for API and OAuth token endpoints (needed by SPA clients)
path := r.URL.Path
if strings.HasPrefix(path, "/api/") || path == "/oauth/token" {
    origin := r.Header.Get("Origin")
    if origin != "" {
        w.Header().Set("Access-Control-Allow-Origin", origin)  // reflects ANY origin
        w.Header().Set("Access-Control-Allow-Methods", "GET, POST, PUT, DELETE, OPTIONS")
        w.Header().Set("Access-Control-Allow-Headers", "Authorization, Content-Type")
        w.Header().Set("Access-Control-Max-Age", "86400")
        w.Header().Set("Vary", "Origin")
    }
}

MEDIUM Timing-Based Username Enumeration CONFIRMED LIVE
internal/service/auth_service.go:61-63

The code attempts to prevent timing-based enumeration by running a dummy bcrypt comparison for unknown users. However, the dummy hash is a placeholder string that doesn't match valid bcrypt format, causing bcrypt to bail out early instead of doing real work.

Vulnerable Code

user, err := s.users.GetByUsername(username)
if errors.Is(err, domain.ErrNotFound) {
    // Constant-time: still run bcrypt to prevent timing-based username enumeration
    auth.CheckPassword(password, "$2a$12$placeholder-hash-to-burn-time-here")
    // ^^^ THIS IS NOT A VALID BCRYPT HASH!
    // bcrypt.CompareHashAndPassword returns immediately with ErrHashTooShort
    // Real hash has 60 chars with proper salt; this is ~42 chars
    s.record(&domain.AuthEvent{...})
    return nil, ErrInvalidCredentials
}

Root cause: The placeholder string $2a$12$placeholder-hash-to-burn-time-here is not a valid bcrypt hash (too short, invalid salt encoding). bcrypt.CompareHashAndPassword returns immediately with an error instead of performing the expensive comparison.

MEDIUM Admin Session Not Validated Against User State CODE REVIEW
internal/handler/admin/handler.go:58-69

Admin sessions are JWT cookies with an 8-hour TTL. The validation function only checks the JWT signature and expiry — it never queries the database to confirm the user still exists, is still active, or still has the admin role.

Vulnerable Code

func (h *adminHandler) mintSession(username string) (string, error) {
    claims := jwt.RegisteredClaims{
        ExpiresAt: jwt.NewNumericDate(time.Now().Add(sessionTTL)),  // 8 hours
        Subject:   username,  // only the username, nothing else
    }
    token := jwt.NewWithClaims(jwt.SigningMethodHS256, claims)
    return token.SignedString([]byte(h.cfg.SessionSecret))
}

func (h *adminHandler) validateSession(tokenStr string) bool {
    _, err := h.parseSession(tokenStr)   // only checks sig + expiry
    return err == nil                     // never queries DB for user state
}

7. Admin Login CSRF CONFIRMED LIVE

POST /admin/login has no CSRF token (see router.go:42 — no requireCSRF wrapper). An attacker can auto-submit a form to log the victim into the attacker's account (login CSRF). 

// router.go:41-42 — login routes have no CSRF middleware
mux.HandleFunc("GET /admin/login", h.loginGet)
mux.HandleFunc("POST /admin/login", h.loginPost)  // no requireCSRF!

// compare with logout which IS protected:
mux.Handle("POST /admin/logout", h.requireSession(h.requireCSRF(...)))

8. Backup Triggered via GET

GET /admin/backup triggers a side-effect (R2 backup). GET requests should be safe/idempotent. A simple <img src="/admin/backup"> on any page the admin visits would trigger a backup. 

mux.Handle("GET /admin/backup", h.requireSession(...))  // should be POST + CSRF

3. What's Done Well

The codebase demonstrates strong security fundamentals. Many common vulnerability classes are properly mitigated:

  • All SQL queries use parameterized statements (? placeholders) — no injection
  • No hardcoded secrets — JWT secrets auto-generated or loaded from env vars
  • Refresh token rotation with family-based theft detection
  • All tokens stored as SHA-256 hashes — raw tokens never persisted to disk
  • bcrypt at cost 12 for password hashing
  • PKCE S256 required for OAuth — no downgrade to plain
  • Auth codes are single-use with atomic check-and-mark in the DB
  • IDOR protection — non-admins can only read their own user record
  • Strong security headers: X-Frame-Options DENY, X-Content-Type-Options nosniff, Referrer-Policy
  • Session cookies: HttpOnly, SameSite=Strict, Secure in production
  • OAuth client_id validated before any redirect — prevents open redirects
  • JWT signing algorithm verified on parse (line 100-101 of jwt.go) — no alg confusion
  • DB file permissions set to 0600
  • Clean JSON error responses — no stack traces or framework details leaked

5. Remediation Plan

Ordered by priority. Effort estimates are relative to this codebase.

Priority 1: Fix CORS — Allowlist Origins

Effort: ~30 minutes  |  Impact: Eliminates the highest severity finding

// cmd/server/main.go — replace the origin reflection with an allowlist

// Add to config:
var allowedOrigins = map[string]bool{
    "https://your-spa.example.com": true,
    "https://admin.example.com":    true,
}

// In the CORS handler:
origin := r.Header.Get("Origin")
if allowedOrigins[origin] {
    w.Header().Set("Access-Control-Allow-Origin", origin)
    // ... rest of CORS headers
}

Better yet, make this configurable via environment variable (CORS_ALLOWED_ORIGINS=https://spa.example.com,https://admin.example.com) so it can be set per deployment.

Priority 2: Add Rate Limiting

Effort: ~2 hours  |  Impact: Blocks brute-force and credential stuffing

// Option A: Use golang.org/x/time/rate for per-IP limiting
// Option B: Use a middleware like github.com/ulule/limiter

// Suggested limits:
//   POST /api/v1/auth/login   : 5 attempts per minute per IP
//   POST /admin/login          : 5 attempts per minute per IP
//   POST /oauth/authorize      : 5 attempts per minute per IP
//   POST /oauth/token           : 10 per minute per IP
//   POST /api/v1/auth/refresh  : 10 per minute per IP
//   Global fallback             : 60 per minute per IP

Consider adding account lockout after N failures (e.g., 10 failed attempts locks for 15 minutes). The OpenAPI spec already defines a rate_limited error code — just need to implement it. Also consider leveraging Cloudflare's built-in rate limiting rules as a first line of defense.

Priority 3: Fix Timing Side-Channel

Effort: ~15 minutes  |  Impact: Eliminates username enumeration

// internal/service/auth_service.go
// Replace the invalid placeholder with a real pre-computed bcrypt hash:

var dummyHash, _ = auth.HashPassword("dummy-password-for-timing", 12)

func (s *AuthService) Login(...) {
    user, err := s.users.GetByUsername(username)
    if errors.Is(err, domain.ErrNotFound) {
        auth.CheckPassword(password, dummyHash)  // now actually runs bcrypt
        ...
    }
}

The current placeholder $2a$12$placeholder-hash-to-burn-time-here is not a valid bcrypt hash (wrong length, invalid salt encoding). bcrypt rejects it immediately, defeating the timing protection. Generate a real hash at startup with the same cost factor.

Priority 4: Validate Admin Sessions Against DB

Effort: ~30 minutes  |  Impact: Prevents stale session abuse

func (h *adminHandler) validateSession(tokenStr string) bool {
    claims, err := h.parseSession(tokenStr)
    if err != nil {
        return false
    }
    // NEW: verify user still exists and is still admin
    user, err := h.userSvc.GetByUsername(claims.Subject)
    if err != nil || !user.Active || user.Role != "admin" {
        return false
    }
    return true
}

This adds one DB query per admin page load. With SQLite and the expected traffic volume, this is negligible. Alternatively, reduce session TTL from 8 hours to 1 hour.

Priority 5: URL-Encode the State Parameter

Effort: ~2 minutes  |  Impact: Prevents URL structure manipulation

// internal/handler/oauth/handler.go:147
// Before:
redirectURL += "&state=" + state

// After:
redirectURL += "&state=" + url.QueryEscape(state)

Priority 6: Additional Hardening

Effort: ~1-2 hours total

  • Add CSRF token to OAuth authorize form and admin login form
  • Change GET /admin/backup to POST with CSRF protection
  • Remove 'unsafe-inline' from CSP script-src — use nonces or move inline scripts to files
  • Add password complexity requirements (or integrate zxcvbn-go for strength estimation)
  • Increase HSTS to max-age=31536000; includeSubDomains; preload
  • Consider restricting /admin/ by IP or moving to an internal network
  • Consider adding Disallow: /admin/ and Disallow: /api/ to a custom robots.txt
